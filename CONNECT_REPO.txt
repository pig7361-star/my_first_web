레포지토리 연결 가이드

개요
- 이 파일은 현재 작업 디렉터리를 원격 Git 리포지토리(예: GitHub)에 연결하고 처음 푸시하는 기본 절차를 설명합니다.

사전 준비
- Git이 설치되어 있어야 합니다: https://git-scm.com/
- (선택) GitHub 계정이 필요합니다.
- (선택) GitHub CLI(`gh`)를 사용하려면 설치하세요: https://cli.github.com/

절차
1) 로컬 저장소 초기화
- 작업 디렉터리에서 다음을 실행합니다:

  git init
  git branch -M main

2) .gitignore 및 README 생성(권장)
- 예: Node 프로젝트일 경우:

  echo node_modules/ > .gitignore
  echo "# 프로젝트 이름" > README.md

3) 파일 추가 및 첫 커밋

  git add .
  git commit -m "Initial commit"

4) 원격 리포지토리 생성 (옵션 A: GitHub 웹)
- GitHub에 로그인 → New repository → 이름 입력 → Create repository
- 생성 후 나오는 원격 URL(HTTPS 또는 SSH)을 복사합니다. 예: https://github.com/username/repo.git

옵션 B: GitHub CLI 사용
- 로그인:

  gh auth login

- 리포지토리 생성 및 원격/푸시 자동 설정:

  gh repo create <repo-name> --public --source=. --remote=origin --push

(예: gh repo create my-repo --public --source=. --remote=origin --push)

5) 원격 추가 및 푸시 (웹에서 생성했을 경우)
- 복사한 원격 URL을 사용:

  git remote add origin <REMOTE_URL>
  git push -u origin main

6) SSH 키 사용 설정(선택)
- SSH를 선호하면 로컬에서 키 생성 및 GitHub에 등록:

  ssh-keygen -t ed25519 -C "your_email@example.com"
  # 공개키 복사
  cat ~/.ssh/id_ed25519.pub

- GitHub Settings → SSH and GPG keys → New SSH key에 붙여넣기
- 원격을 SSH URL로 변경 (필요 시):

  git remote set-url origin git@github.com:username/repo.git

추가 팁
- 기본 브랜치를 `main`으로 변경하려면 `git branch -M main` 사용.
- 협업 전 `README.md`, `LICENSE` 파일을 추가하세요.
- GUI 도구(예: GitHub Desktop, SourceTree)를 사용하면 초기 설정이 쉬워집니다.

문제 해결
- 인증 오류(HTTPS): 캐시된 자격증명 또는 토큰 문제일 수 있습니다. `gh auth login` 또는 Personal Access Token 사용을 고려하세요.
- 권한 오류(푸시 실패): 리포지토리 소유자/권한을 확인하세요.

- [ ] `git remote add origin` 및 `git push -u origin main`
